### 13. 后空翻（back_flip）

| 项目 | 内容 |
|------|------|
| 技能 | `do_dog_behavior` |
| 动作类型 | 狗行为（dog_behavior） |
| 行为名称 | `back_flip` |
| 调用结果 | **成功（status=3 SUCCEEDED, result=1）** |
| 说明 | 后空翻。通过 `do_action`（action_id=260）直接调用时失败（result=3），改用 `do_dog_behavior` 成功 |

**调用代码（Python actionlib）：**

```python
cli = actionlib.SimpleActionClient('/agent_skill/do_dog_behavior/execute', ExecuteAction)
goal.args = json.dumps({"behavior": "back_flip"})
```

**调用代码（rostopic pub）：**

```bash
rostopic pub --once /agent_skill/do_dog_behavior/execute/goal agent_msgs/ExecuteActionGoal "{header: {seq: 0, stamp: {secs: 0, nsecs: 0}, frame_id: ''}, goal_id: {stamp: {secs: 0, nsecs: 0}, id: ''}, goal: {invoker: 'cli', invoke_priority: 50, hold_time: 15.0, args: '{\"behavior\":\"back_flip\"}'}}"
```

**实测 result 输出：**

```yaml
status:
  status: 3                  # actionlib SUCCEEDED
  text: ''
result:
  result: 1                  # 业务层成功
  response: ''
```

**备注：**
- 通过 `do_action` 调用 Back flip（260）时，优先级 30 被拒绝（reason: low priority），优先级 50 执行失败（result=3）
- 改用 `do_dog_behavior` 的 `back_flip` 行为可以稳定执行
- `invoke_priority` 必须 ≥ 50 才能抢占低优先级技能槽位
- 后空翻是高危动作，请确保周围有足够空间

#### 13.1 深入诊断：`do_action` 调用 `action_id=260` 的真实失败原因

后续以 `invoke_priority: 50` 重新发送 `do_action/action_id=260`，第一次拿到的真实拒绝原因是：

```yaml
status: 4                                     # ABORTED
text: "Current body height is too high to jump!"
result: 3
```

也就是说裸的 `do_action 260` **不会自动压低身体**，而 `do_dog_behavior/back_flip` 内部其实串了"压低 → 蓄力 → 起跳"完整流水线。

#### 13.2 手动模拟流水线：prepare_flip_x（20742）+ Back flip（260）

| 步骤 | 调用 | 结果 |
|---|---|---|
| ① | `do_action action_id=20742`（`prepare_flip_x` 准备前/后翻） | ✅ status=3, result=1 |
| ② | `do_action action_id=260`（`Back flip` 后空翻） | ❌ status=4, text="Not ready to jump again!", result=3 |

第二步失败说明**跳跃类动作存在冷却保护机制**——刚刚通过 `do_dog_behavior/back_flip` 起跳过一次，电机/IMU 进入冷却恢复期，必须等待若干秒才能再次起跳。

**结论与最佳实践：**

- **后空翻请始终使用 `do_dog_behavior` + `behavior=back_flip`**，它会自动处理：
  - 身高判定 / 自动压低（隐式 `prepare_flip_x`）
  - 优先级抢占
  - 冷却时序
- 如果坚持用 `do_action 260` 实现底层调用，必须自行串联：
  1. `do_action 20742` (prepare_flip_x)
  2. 等待 ≥ 60s 冷却（如果不久前刚跳过）
  3. `do_action 260` (Back flip)
- 可类比的预备动作：`prepare_flip_y`（20743）用于左/右空翻（262/263）

## 调用模板

所有动作通过以下统一模板调用：

```python
import json, rospy, actionlib
from agent_msgs.msg import ExecuteAction, ExecuteGoal

rospy.init_node('action_node', anonymous=True)

# do_action 技能（基础动作 + 扩展动作）
cli = actionlib.SimpleActionClient('/agent_skill/do_action/execute', ExecuteAction)
cli.wait_for_server(timeout=rospy.Duration(3.0))

goal = ExecuteGoal()
goal.invoker = 'my_app'
goal.invoke_priority = 30
goal.hold_time = 10.0
goal.args = json.dumps({'action_id': <动作ID>})

cli.send_goal(goal)
cli.wait_for_result(timeout=rospy.Duration(30.0))
result = cli.get_result()
print(f'Result: {result.result}')  # 1=成功
```

```python
# do_dog_behavior 技能（狗行为）
cli = actionlib.SimpleActionClient('/agent_skill/do_dog_behavior/execute', ExecuteAction)
cli.wait_for_server(timeout=rospy.Duration(3.0))

goal = ExecuteGoal()
goal.invoker = 'my_app'
goal.invoke_priority = 30
goal.hold_time = 10.0
goal.args = json.dumps({'behavior': '<行为名称>'})

cli.send_goal(goal)
cli.wait_for_result(timeout=rospy.Duration(30.0))
result = cli.get_result()
print(f'Result: {result.result}')  # 1=成功
```

## 经验总结

1. **前进移动**不能通过直接发 `set_velocity` 话题实现，需要使用 `step_forward`（20524）或 `joy_walk`（20564）等扩展动作
2. **匍匐（crawl）**行为只是姿态，不包含移动
3. 从匍匐/趴下状态恢复需要先执行 **Recovery stand**（action_id=3）
4. 基础动作和扩展动作都通过 `do_action` 技能的 `action_id` 调用
5. 狗行为通过 `do_dog_behavior` 技能的 `behavior` 名称调用
6. **空翻类动作**（260/261/262/263）必须用 `do_dog_behavior` 调用对应 behavior，否则需自行串 `prepare_flip_x/y` 并处理冷却
7. **`invoke_priority` ≥ 50** 才能稳定抢占被其它客户端占用的技能槽
8. `result` 话题不是 latched，需要先 `rostopic echo` 监听再发布 goal 才能捕获结果

## 通用调用脚本

仓库已封装了两段命令行工具，避免每次手撸长 YAML：

| 脚本 | 位置 | 说明 |
|---|---|---|
| `dog_cmd.sh` | [`scripts/dog_cmd.sh`](../scripts/dog_cmd.sh) | 在狗内部执行的统一入口（封装 `rostopic pub`） |
| `remote_dog.sh` | [`scripts/remote_dog.sh`](../scripts/remote_dog.sh) | 开发机端 SSH 封装，自动 source ROS 环境并转发参数 |

**典型用法：**

```bash
# 一次性把脚本同步到狗
./scripts/remote_dog.sh sync

# 触发动作 / 行为
./scripts/remote_dog.sh action 515                 # 敲门
./scripts/remote_dog.sh action 20524 30 5          # step_forward, prio=30, hold=5s
./scripts/remote_dog.sh behavior back_flip 50 15   # 后空翻
./scripts/remote_dog.sh behavior crawl

# 监听最近一次 result（建议另开一个终端先监听再发 goal）
./scripts/remote_dog.sh result action
./scripts/remote_dog.sh result behavior

# 直接进入狗 shell
./scripts/remote_dog.sh shell
```

环境变量可覆盖：`DOG_HOST`（默认 `root@10.10.10.10`）、`DOG_PASS`（默认 `weilan.com`）。
