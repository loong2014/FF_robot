# 两周 MVP 排期版功能清单

> 更新时间：2026-04-25
> 适用范围：当前两周交付版本
> 长期规划：互联、动作控制、教育、视觉、SKILL 平台
> 当前 MVP 范围：互联 + 动作控制 + 部分教育
> 当前 MVP 不包含：视觉、SKILL 平台

## 1. 范围说明

### 1.1 长期规划

- 互联：App、BLE、TCP、MQTT、`robot_server`、ROS 的通信与连接
- 动作控制：姿态控制、连续控制、腿部参数控制、步态参数控制
- 教育：面向 K12 的图形化编程
- 视觉：通过外置摄像头实现人物识别、手势识别和跟随
- SKILL 平台：用户通过平台自定义动作，而不是由研发持续手工新增自定义动作

### 1.2 当前已实现

- 已打通 `App -> BLE -> robot_server -> ROS -> 机器狗` 控制主链
- 已具备 BLE 扫描、连接、状态展示和基础动作控制能力

### 1.3 当前 MVP 目标

当前两周版本只交付：

- 互联
- 动作控制
- 部分教育

不纳入当前版本：

- 视觉能力
- SKILL 平台
- 完整课程模板系统
- 高级 Blockly 工作台
- 复杂控制流和高级编辑器
- 参数化动作编辑器

## 2. 排期版功能清单

| 功能项 | 优先级 | 依赖 | 验收方式 | 建议阶段 |

## 3. 建议排期拆分

### 第 1 周必须完成

- BLE 连接与控制主链稳定
- Blockly / Block MVP 容器
- 站立 / 坐下 / 停止 / 移动 block
- 基础状态展示

### 第 2 周必须完成

- block 交互收口
- 错误提示
- 默认课堂演示程序
- 演示文档与真机回归

## 4. 优先级规则

- `P0`：两周版本必须交付
- `P1`：第 2 周优先做；若时间不足，可压缩交互复杂度，但不能完全缺席
- 不排入本次：复杂控制流、模板系统、导入导出、完整高级编辑器、MQTT 深化、视觉能力、SKILL 平台、参数化动作编辑器

## 5. 两周 MVP 验收口径

- 教师能完成连接、拖拽 block、执行课堂演示
- 坐下、站立、停止、移动可稳定执行
- 参数非法时不会直接下发到机器人
- 演示过程中可随时停止

## 6. 当前版本外的长期规划

以下方向保留在后续阶段：

- 完整教育能力：高级 Blockly、模板系统、课程组织
- 动作控制深化：参数化动作编辑器、自定义动作能力
- 视觉能力：外置摄像头接入、人物识别、手势识别、跟随
- SKILL 平台：用户自定义动作平台与管理能力
